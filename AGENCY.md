### Méthodes de la Division 16

```
LINK BUILDING PLAYBOOK — 8 STRATÉGIES PARALLÈLES

1. COMPETITOR GAP ANALYSIS
   → Reverse-engineer top 5 concurrents
   → Identifier TOUS les domaines qui linkent vers eux mais pas vers nous
   → Outreach ciblé avec contenu supérieur

2. SKYSCRAPER METHOD (Brian Dean)
   → Trouver le contenu #1 sur chaque keyword cible
   → Créer une version 10x meilleure (plus data, plus visuel, plus récent)
   → Contacter tous ceux qui linkent vers l'original

3. BROKEN LINK BUILDING
   → Scanner les sites d'autorité de la niche
   → Identifier les liens cassés (404/410)
   → Proposer notre contenu en remplacement — win-win

4. DIGITAL PR & DATA STORIES
   → Études originales avec données exclusives
   → Infographies partageables
   → Pitchs journalistes (HARO, Connectively, direct)
   → Newsjacking sur événements trending

5. GUEST POSTING PREMIUM
   → Cibler UNIQUEMENT DA50+ avec traffic réel
   → Contenu expert, pas du spam
   → Bio + contextual links naturels

6. RESOURCE PAGE LINK BUILDING
   → Miner les pages "Best resources for X"
   → Soumettre pour inclusion
   → Link reclamation sur brand mentions sans lien

7. LINKABLE ASSETS
   → Outils gratuits (calculateurs, checkers, generators)
   → Templates téléchargeables
   → Études annuelles avec données propriétaires
   → Widgets embeddables

8. NICHE EDITS & CONTEXTUAL INSERTIONS
   → Identifier articles existants pertinents sur sites DA40+
   → Proposer un ajout contextuel naturel
   → Anchor text optimisé mais naturel
```

## RÈGLES D'OR

1. **Un brief = un déploiement** — pas de questions superflues, on exécute
2. **Parallélisme maximal** — tout ce qui peut tourner en même temps, tourne
3. **Quality gate obligatoire** — rien ne sort sans Reality Checker + Evidence Collector
4. **Data-driven** — chaque décision appuyée par des données, pas des opinions
5. **Score minimum** — pas de livrable en dessous du seuil qualité défini par le client
6. **Feedback loop** — chaque campagne nourrit la suivante via Experiment Tracker
7. **Budget-aware** — Financial Analyst valide chaque allocation avant exécution
